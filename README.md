## 📌 Описание проекта
Задача проекта — бинарная классификация исхода матча в Dota 2 (победа или поражение сил Света, столбец `radiant_win`) на основе анонимизированных игровых событий, собранных через OpenDota API. Главная особенность — модель делает предсказание, используя данные **строго за первые 15 минут** после начала матча.

Целевая метрика соревнования — **коэффициент Джини (Gini)**, который рассчитывается на основе ROC-AUC:
$$\text{Gini} = 2 \cdot \text{ROC-AUC} - 1$$

### 🧠 Ключевые этапы разработки

#### 1. Разведочный анализ (EDA) и очистка данных

* **Анализ распределений:** Выявлен Data Drift в признаке `region` между обучением и тестом (применен сглаженный `TargetEncoder`) и обнаружен слом тренда винрейта 23 мая 2024 года, (вероятно) вызванный патчем 7.36.
* **Трансформация признаков:** Для рейтинга игроков (фича `avg_mmr`) применено логарифмирование ($\log(1 + x)$) для приведения логнормального распределения к нормальному.
* **Фильтрация строк датасета:** Из таблицы `player_df` удалены аномальные матчи (некорректный состав команд, скрытые герои, невалидные аккаунты).

#### 2. Feature Engineering

* **Heroes Sign Encoding:** Создан кастомный `HeroesEncoder`, преобразующий состав команд в разреженный вектор по принципу: `1` — Radiant, `-1` — Dire, `0` — не пикнут.
* **Агрегация временных рядов:** Поминутные графики золота и опыта за первые 15 минут агрегированы через базовые статистики: `mean`, `std`, `max_lead` +  количество смен лидера: `lead_switches`.
* **Анализ трендов (OLS):** С помощью МНК (OLS) извлечены коэффициенты угла наклона (`slope`) и сдвига (`intercept`), описывающие динамику и темп преимущества команд.
* **Бинаризация:** Реализовано локальное разбиение 15 минут игры на бины (`BinStatisticExtractor`) для локализации фичей на каждом временном интервале.

#### 3. Архитектура пайплайна и оптимизация

* **Инженерия конвейера:** Логика обработки данных объединена в сквозной пайплайн на базе `ColumnTransformer` и `FeatureUnion`.
* **Моделирование и тюнинг:** В качестве базового алгоритма выбрана логистическая регрессия. Оптимизация гиперпараметров проводилась через `Optuna`.
* **Финальное решение:** Модель с L2-регуляризацией (`penalty="l2"`, `solver="lbfgs"`, `C=0.86942`) обучена на всей выборке. Её предкты на тествовой выборке были посланы на Kaggle. Итоговый **Score (aka Gini): 0.38666**.
